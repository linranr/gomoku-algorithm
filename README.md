# 五子棋在线对弈平台

基于Python Flask和Web技术开发的五子棋游戏，支持人机对战和好友对战，提供三种AI难度可选。

## 快速开始

### 环境要求
- Python 3.8+（推荐3.13.12）
- 现代浏览器（Chrome/Firefox/Safari/Edge）

### 安装依赖
```bash
pip install flask flask-cors pygame
```

### 启动服务器
```bash
python main.py
```

**注意**：服务器默认使用80端口，需要管理员权限运行。如果启动失败，请以管理员身份运行程序。

### 访问游戏
- 本地访问：`http://localhost`
- 局域网访问：`http://[你的IP地址]`
- 直接打开：浏览器打开 `frontend/index.html`

## 功能特性

### 游戏模式
- **人机对战**：三种AI难度可选
  - 简单：随机策略，适合入门
  - 普通：贪心算法，有一定挑战性
  - 困难：Minimax+α-β剪枝，搜索深度4层
  简单难度在随机落子的基础上，增加了对玩家三连子（两边为空）和四连子的拦截，保证AI具备基本防守能力。
- **好友对战**：
  - 本地双人对弈
  - 与好友在线对弈（规划中...）

### 游戏功能
- 悔棋：撤销玩家和AI各一步
- 重新开始：重置棋盘重新开始
- 认输：主动认输结束游戏
- 落子音效：播放落子时的音效
- 棋谱导出：导出对局记录为PNG图片
- 回合提示：显示当前轮到哪方出棋
- 胜负判定：自动判断游戏胜负

### 界面特点
- 响应式设计：支持PC/平板/移动端
- 明暗主题：支持主题切换
- 简洁美观的UI风格
- 流畅的交互动画

### 趣味登录功能
**说明**：本功能为**前端交互演示模块**，通过模拟登录/注册流程，展示页面跳转、过渡动画与趣味交互设计，不依赖真实后端与数据库，旨在提升用户体验与项目趣味性。

#### 1. 页面跳转与过渡效果
- **首页到登录界面**：用户点击首页登录按钮后，系统实现平滑过渡动画跳转至登录界面
- **登录到注册界面**：登录界面点击"注册"按钮时，平滑跳转至注册界面
- **注册到登录界面**：注册界面点击"返回登录"按钮时，平滑跳转回登录界面
- **视觉效果**：所有页面间的切换均添加视觉过渡动画，确保流畅的用户体验

## API接口

| 接口 | 方法 | 说明 | 请求参数 |
|------|------|------|----------|
| `/api/move` | POST | 落子 | `row`, `col`, `difficulty` |
| `/api/restart` | POST | 重新开始 | `first_hand` |
| `/api/undo` | POST | 悔棋 | 无 |
| `/api/resign` | POST | 认输 | 无 |

### 接口详情

#### 落子接口 `/api/move`
**请求参数**：
- `row` (int): 落子行号（0-14）
- `col` (int): 落子列号（0-14）
- `difficulty` (string): AI难度（'easy'/'normal'/'hard'）

**返回数据**：
```json
{
  "board": [[0,0,...],...],
  "current_player": 1,
  "player_color": 1,
  "ai_color": 2,
  "game_over": false,
  "winner": null
}
```

#### 重新开始接口 `/api/restart`
**请求参数**：
- `first_hand` (int): 先手选择（1=玩家先手, 2=AI先手）

**返回数据**：
```json
{
  "board": [[0,0,...],...],
  "current_player": 1,
  "player_color": 1,
  "ai_color": 2,
  "game_over": false,
  "winner": null
}
```

#### 悔棋接口 `/api/undo`
**请求参数**：无

**返回数据**：
```json
{
  "board": [[0,0,...],...],
  "current_player": 1,
  "player_color": 1,
  "ai_color": 2,
  "game_over": false,
  "winner": null
}
```

#### 认输接口 `/api/resign`
**请求参数**：无

**返回数据**：
```json
{
  "board": [[0,0,...],...],
  "current_player": 1,
  "game_over": true,
  "winner": 2
}
```

## AI算法说明

### 简单难度
- 策略：随机落子，带基本围堵功能
- 特点：优先检测并围堵玩家4连子和3连子威胁
- 适用：初学者入门

### 普通难度
- 策略：贪心算法
- 特点：综合评估进攻和防守得分，选择最优位置
- 适用：有一定基础的玩家

### 困难难度
- 策略：Minimax算法配合α-β剪枝
- 搜索深度：4层
- 优化：Zobrist哈希表、候选位置剪枝
- 适用：有挑战需求的玩家

## 常见问题

### 启动问题
**Q：启动失败，提示端口被占用？**
A：80端口需要管理员权限，请以管理员身份运行程序。

**Q：浏览器无法访问？**
A：检查防火墙设置，确保80端口未被阻止。

### 游戏问题
**Q：落子音效无法播放？**
A：确保浏览器允许自动播放音效，或检查音效文件路径是否正确。

**Q：AI响应很慢？**
A：困难难度AI需要较多计算时间，这是正常现象。可以降低难度或等待计算完成。

### 兼容性问题
**Q：移动端显示异常？**
A：项目支持响应式设计，如遇显示问题请尝试刷新页面或更换浏览器。
